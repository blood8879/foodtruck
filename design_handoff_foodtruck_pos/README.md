# Handoff: 푸드트럭 POS · 메뉴 · 매출 관리 앱

푸드트럭 사장님용 메뉴·매출 관리 앱. 장사 중엔 빠르게 주문을 찍고(POS), 사장은 일/월/연 매출과 순이익을 확인한다. **모바일 앱**(현장 입력) + **PC 웹**(조회·분석), 오프라인 동작 전제.

---

## 이 디자인 파일에 대하여 (먼저 읽어주세요)

이 번들의 파일은 **HTML로 만든 디자인 레퍼런스**입니다. 의도한 모양과 동작을 보여주는 프로토타입이지, 그대로 가져다 쓰는 프로덕션 코드가 아닙니다.

당신의 작업은 이 HTML 디자인을 **대상 코드베이스의 기존 환경**(React Native / Flutter / SwiftUI / 웹 React 등)에서, 그 환경의 확립된 패턴·라이브러리로 **재현**하는 것입니다. 아직 환경이 없다면, 이 프로젝트에 가장 적합한 프레임워크를 골라 구현하세요.

- 모바일 화면 = 휴대폰 베젤은 디자인 레퍼런스용 목업입니다. 실제 구현 시 베젤은 빼고 화면 콘텐츠만 구현하세요.
- PC 웹 화면 = 브라우저 창 크롬(신호등·주소창)은 목업입니다. 실제로는 그 안의 대시보드 UI만 구현합니다.

## 충실도 (Fidelity)

**High-fidelity (하이파이).** 최종 색상·타이포그래피·간격·잠금(게이팅) 표시까지 의도대로 반영된 목업입니다. 색/폰트/간격을 아래 **디자인 토큰**과 각 화면 명세에 맞춰 픽셀에 가깝게 재현하되, 컴포넌트 구현은 대상 코드베이스의 기존 라이브러리/패턴을 사용하세요.

## 화면 명세

모바일 프레임 콘텐츠 영역 = **368×752px**(390×844 베젤에서 보더 11px 제외). 상태바 46px, 하단 탭바 82px.

### 사장 모바일 — 하단 탭 4개

**① 장사 시작 (영업 세션 open)** — POS 진입 전 상태
- 상단: 날짜(좌) + "영업 전" 배지(우, 회색 점).
- 중앙 히어로: 트럭 아이콘(84×84 라운드 카드) → 상호명(24/800) → "오늘 아직 영업을 시작하지 않았어요" → 어제 매출/어제 순이익 2분할 카드.
- 하단: 큰 `장사 시작` 버튼(62px, accent, play_circle). 아래 캡션 "무료 플랜 · 시작 시 전면 광고 1회 노출".
- 동작: 탭하면 영업 세션 open → (무료) 전면 광고 1회 → POS 진입. 종료 시 `장사 종료`로 세션 close + (무료) 광고 1회.

**② 주문 (POS) — 장사 중 메인**
- 상단바: "영업 중 · 02:14" 라이브 배지(accent 점) + "온라인 · 대기 0" 동기화 배지(green, cloud_done). 오프라인이면 "대기 N건".
- 카테고리 칩: 버거/음료/사이드/세트 가로 스크롤, 활성 칩 다크.
- 메뉴 그리드: 2열 카드(min-h 92). 카드 = 이름 + 가격. 장바구니에 담긴 수량은 우상단 원형 배지(accent). **품절** 메뉴는 흐림(opacity .55) + "품절" 배지 + 비활성.
- 장바구니 시트(하단 고정, 탭바 위): 핸들 + "장바구니 N" + 항목 행[이름 · −/수량/+ 스테퍼 · 라인합]. 할인메모 행("현금 -500"). **총액 수동 조정**(tune 아이콘). 큰 `주문 완료` 버튼.
- 하단 탭바: 주문(활성)/매출/메뉴/설정.

**③ 매출 — 사장 핵심 조회 (위→아래 순서 고정)**
1. **큰 숫자 카드**: 오늘 매출(다크 카드, 흰 숫자, "▲12% 어제比") / 오늘 순이익(green). 상단 우측 **일·월·연 토글** — 일=활성, 월·연=잠금 아이콘(gold lock, 유료).
2. **기간별 추이 그래프** — **유료 잠금**: 막대가 흐림(blur 3px) 처리되고 그 위 프로스티드 오버레이 + 자물쇠 원형 + "유료 플랜에서 추이 그래프 잠금 해제" + `업그레이드`(gold) 버튼. 카드 헤더에 "유료" 배지.
3. **메뉴별 판매 TOP**: 순위·이름·수량·매출액 + 진행 막대(매출액 기준).
4. **주문 내역 리스트**: 시각·요약·입력자(사장/알바)·금액. **취소건** = 취소 배지 + 취소선. **지각 동기화분** = 주황 "지각 동기화" 배지.

**④ 메뉴 관리**
- 헤더: "메뉴 관리" + `+ 추가`(다크).
- 카테고리별 그룹 리스트(버거 / 사이드·음료). 행 = 이름 + 가격 + **원가율 칩**(≤40% green, >40% gold) + **품절 토글 스위치**. 품절 항목은 흐림 + "· 품절".

**⑤ 메뉴 추가 · 수정 (서브 화면, 탭바 없음)**
- 헤더: 뒤로(arrow_back) + "메뉴 수정" + `저장`(accent).
- 필드: 이름 / 판매가·원가(2분할) / **원가율 자동 계산 배지**(green, "마진 ₩4,650") / 카테고리 칩 / 품절 토글 / **레시피=재료 리스트**(재료·단가·수량 행 + `재료 추가`). 재료 단가 합산으로 원가 자동 산출.

**⑥ 설정**
- 트럭 정보 카드(아이콘 + 상호 + 사장명 + chevron).
- **직원 관리**: 직원 행(아바타 + 이름 + "주문 전용" 배지 + PIN) + `직원 초대 · 초대코드/PIN 발급`. (무료에서도 다중 직원 OK 표기)
- **요금제**(다크 카드): "현재 · 무료 플랜" + 잠금 기능 목록(기간 분석/추이 그래프/PC 웹/데이터 내보내기/광고 제거, 각 gold lock) + `유료 업그레이드`(gold) 버튼.

### 알바 모바일

**⑦ 알바 POS — 주문 탭 하나만**
- ② POS와 동일한 주문 UI. 단 **하단 탭바 없음**(다른 탭 숨김). 상단 좌측에 "알바 모드 · 김알바" 배지(staff 보라). 매출/원가/관리 일절 노출 안 됨.

### 세션 · 광고

**⑧ 전면 광고 — 장사 시작/종료 시 (무료만)**
- 전체 다크 화면. 좌상단 "광고 · AD" 라벨, 우상단 "건너뛰기 4" 카운트다운 칩(비활성). 중앙 스트라이프 플레이스홀더("광고 크리에이티브 320×480"). 하단 캡션 "장사 시작 전면 광고 · POS·매출·알바 화면엔 광고 없음 · 유료 전환 시 제거".

### 사장 PC 웹 (모두 유료 플랜 전용 · 1420×900 목업)

좌측 고정 사이드바 230px(다크): 로고 + 대시보드/매출 분석/메뉴 관리/직원/설정 + 하단 "유료 플랜" 카드. 메인은 `#F6F2EC` 배경, 패딩 24×28.

**⑨ 대시보드** — 헤더(기간 토글 일/주/월/연, 날짜 범위, `내보내기`) + KPI 4카드(총매출 다크 / 순이익 / 주문수 / 객단가) + 좌: 일별 매출 추이 막대 그래프, 우: 메뉴별 TOP + 최근 주문(취소/지각 배지).

**⑩ 매출 분석** — 일별 매출·순이익 큰 막대 그래프 + 요일별 평균/시간대별 매출 2분할 + **메뉴별 상세 표**(메뉴·판매수량·매출·원가·순이익·원가율).

**⑪ 메뉴 관리** — `+ 메뉴 추가` + 카테고리 필터 칩 + **표**(메뉴·카테고리·판매가·원가·원가율·품절 토글·편집). 품절행 흐림.

**⑫ 직원** — `직원 초대` + 좌: 직원 표(이름·역할·PIN·권한·마지막 접속), 우: **초대 카드**(다크) — 초대코드(9F4K2) + 신규 PIN(refresh) + `초대코드 복사`. 사장=전체, 알바=주문 전용.

**⑬ 설정** — 트럭 정보 폼(상호/사장/연락처/영업지 + 저장) + 데이터(내보내기 CSV·Excel, 오프라인 동기화 상태) + **요금제 카드**(다크, ₩9,900/월, 다음 결제일, 기능 체크리스트, 결제 관리). PC 웹은 유료 전용이므로 기능이 모두 활성(체크) 상태로 표시.

## 핵심 데이터 모델 (화면 필드 근거)

- **메뉴(Menu)**: `id, 이름, 판매가, 원가, 원가율(=원가/판매가, 파생), 카테고리, 품절여부, 레시피?[{재료, 단가, 수량}]`. 원가는 레시피 재료 단가 합으로 산출 가능.
- **주문(Order)**: `id, 일시, 입력자(owner|staff), 라인[{메뉴, 수량, 박제_판매가, 박제_원가}], 총액, 할인메모, 상태(완료|취소), 동기화상태(synced|late)`. 가격·원가는 주문 시점 값으로 **박제(snapshot)** — 이후 메뉴 단가가 바뀌어도 과거 주문은 불변.
- **영업세션(Session)**: `id, 시작시각, 종료시각`. 장사 시작/종료가 **영업일 경계**. 무료 플랜은 open/close 시 각각 광고 1회.
- **집계(Aggregate)**: 일/세션/월/연 단위 — 매출, 원가, **순이익(=매출 − 재료원가)**.
- **직원(Staff)**: `이름, 역할(owner|staff), PIN, 초대코드, 권한, 마지막접속`.

## 상태 / 동작

- **세션 게이트**: POS는 세션 open 상태에서만 주문 입력. 진입 화면(①)에서 시작 → POS, 종료 시 요약 후 close.
- **장바구니**: 메뉴 탭 → 수량 누적(±). 총액 = Σ(수량×판매가) − 할인. 총액 수동 조정 + 할인메모 별도 보관. 주문 완료 시 라인별 단가·원가 박제.
- **품절 토글**: 메뉴 관리(④/⑪)에서 토글 → POS 그리드에서 즉시 비활성/흐림 반영.
- **오프라인 동기화**: 오프라인에서 주문 로컬 저장 → "대기 N건". 재연결 시 업로드, 늦게 올라온 주문은 매출 내역에 "지각 동기화" 배지.
- **추이 그래프 / 월·연 토글 / PC 웹 / 데이터 내보내기**: 무료에서 잠금(gold lock + 라벨), 유료 해제.

## 무료 / 유료 경계 (UI 게이팅 — 잠금 표시 일관 적용)

- **무료**: 메뉴 관리, POS 주문, 다중 직원(알바 동시 주문), 당일·기본 매출 조회.
- **유료(구독, ₩9,900/월)**: 기간 분석(월/연), 추이 그래프, PC 웹 조회, 데이터 내보내기, 광고 제거.
