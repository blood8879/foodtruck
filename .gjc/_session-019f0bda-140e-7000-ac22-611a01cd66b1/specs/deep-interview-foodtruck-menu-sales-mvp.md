- 메뉴 원가 = 직접 입력 기본 + 선택적 레시피(원가계산용; 재고자동차감은 v2) (R1,R8)

- 사장·알바 여러 기기 동시 입력 → 병합 필요 (R6)
- 주문 = append-only 불변 이벤트, 주문라인에 판매가·원가 박제; 메뉴가격 변경 과거 소급 안됨 (R7)

- 일/월/연 매출 자동 재집계, 지각 동기화분 자동합산+표시, lock 없음 (R11)

## Goal
푸드트럭 사장님이 메뉴(판매가·원가·원가율·카테고리·품절토글)를 관리하고, 장사 중 사장·알바가 여러 폰으로 동시·오프라인으로 메뉴버튼을 탭해 주문을 찍으면(취소·할인 포함, 주문은 불변 이벤트+당시 가격·원가 박제), 그 기록이 동기화돼 사장이 PC·폰에서 일/월/연별 매출과 순이익(=매출−재료원가)을 확인하는 모바일+웹 앱. 재고·위치·결제연동은 MVP 제외.

## Constraints
- 모바일(현장 입력)+PC 웹(조회). 오프라인 필수, 연결 시 동기화.
- 다중 사용자 동시 입력 → 충돌 없는 병합.
- 주문=append-only 불변 이벤트 + 가격·원가 박제, 취소=보상 이벤트.
- 권한: 사장(전체)/알바(주문·취소만) 2단계.
- 순이익 = 매출−재료원가만.

## Non-Goals
- 재고 관리·레시피 재고자동차감 (v2)
- 실시간 위치, 외부 결제 연동, 결제수단별 매출 분리 (향후)
- 세무 신고용 매출(부가세·현금영수증), 시재대조 마감정산, 할인 종류별 통계 (비포함)

## Screens & UX
### 사장 모바일 (하단 4탭)
1. **주문(POS)**: 메뉴 버튼 그리드(카테고리별) + 장바구니(수량·총액 수동조정·할인메모) + [주문 완료]. 품절 메뉴 비활성.
2. **매출**: ①오늘 매출·순이익 큰 숫자(일/월/연 토글) ②기간별 추이 그래프 ③메뉴별 판매 순위 TOP ④주문 내역 리스트(취소 포함, 지각반영 표시).
3. **메뉴 관리**: 메뉴 추가/수정(판매가·원가·카테고리·선택레시피), 품절 토글.
4. **설정**: 직원 관리(초대코드/PIN 발급), 트럭 정보.
### 알바 모바일
- 주문(POS) 화면만 노출, 나머지 탭 숨김.
### 사장 PC 웹
- 매출·분석 중심(큰 그래프), 메뉴·설정 조회·편집 가능.
