# 오늘장사 — 디자인 의뢰서 (작업 우선순위 포함)

작성일: 2026-07-08 · 기능 명세: `docs/SPEC.md` (필독) · 기존 시안: `design_handoff_foodtruck_pos/`
의뢰 범위: 모바일 앱(Android) UI 개선 및 신규 화면 디자인. 아래 우선순위 순서대로 착수 요청.

---

## A. 모든 작업에 공통 적용되는 규칙

1. **뷰포트**: 모바일 360~480px 기준. 하단 탭바·안드로이드 내비게이션 영역 고려.
2. **디자인 시스템**: 기존 토큰을 기반으로 개선 (SPEC §9)
   - 배경 `#F6F2EC`(웜 오프화이트), 액센트 `#E85D3A`, 프로/골드 계열, 카카오 `#FEE500`
   - 서체 Pretendard, 금액·숫자는 tabular-nums 고정폭, 아이콘 Material Symbols Rounded
   - 공용 컴포넌트(Card, AppButton, Badge, Chip, LockChip, MoneyText, QtyStepper)를 유지·확장하는 방향. 신규 컴포넌트는 명세와 함께 제안
3. **수익화 UX 제약 (SPEC §6 — 위반 불가)**: 영업 중 전면광고 0회 / POS 화면 광고 금지 / 네이티브 광고는 "광고" 라벨 필수·실패 시 자리 소멸(스켈레톤 금지) / 보상형은 자발 선택만 / 모든 부가 기능은 실패 시 조용히 생략(로딩 블로커 금지)
4. **상태 변형 필수 포함**: 각 화면당 ① 요금제 3종(무료/체험중/프로) ② 빈 상태 ③ 오프라인(동기화 대기 pill) ④ 역할(사장/직원, 해당 시)
5. **산출물 형식**: 와이어프레임 → 하이파이 시안 → **인터랙티브 목업(Figma 프로토타입)** + 컴포넌트/토큰 명세. 개발 핸드오프 가능한 형태(스페이싱·상태별 스펙 포함)
6. **카피라이팅**: 기존 앱 톤 유지 — 존댓말·짧은 문장("~해요체"), 장사 용어(시재, 마감, 품절) 그대로 사용

## B. 작업 우선순위

### 🥇 1순위 — 페이월 "오늘장사 프로" (SPEC §5.14)
> **의뢰문**: SPEC §5.14·§6·§9 기준으로 구독 페이월 화면을 360~480px 인터랙티브 목업으로 디자인해 주세요. 연간(기본 선택, "2개월 무료" 배지)/월간 상품 선택 인터랙션, 구매 복원, Google Play 고지 포함. 상태 3종(결제 미설정 "준비 중" / 가격 로딩 / 로드 실패)도 함께.
- **문제**: 현재 기능만 있는 수준. 전환율을 좌우하는 화면인데 가치 전달·비교·신뢰 요소가 약함
- **완료 기준**: 유료 기능 6종의 가치가 3초 안에 읽힘 / 연간 상품으로 자연 유도 / 해지·청구 고지 가독성
- **참고**: 잠금 → 체험(보상형 광고) → 페이월로 이어지는 퍼널의 종착지임을 감안

### 🥈 2순위 — 매출 탭 재구성 (SPEC §5.6)
> **의뢰문**: SPEC §5.6 기준으로 매출 탭의 정보 위계를 재설계해 주세요. 현재 8개 섹션(KPI/추이/인사이트 진입/메뉴 TOP/지출/주문내역 등)이 세로로 나열돼 과밀합니다. 요약→상세 위계, 기간 토글(오늘/월/연 + 유료 잠금·체험 버튼), 네이티브 광고 슬롯(주문내역 5번째 뒤, "광고" 라벨) 위치를 포함한 인터랙티브 목업으로.
- **완료 기준**: 마감 직후 "오늘 얼마 벌었나(지출 차감 순이익)"가 첫 화면에서 읽힘 / 스크롤 2회 안에 전 섹션 도달 / 무료 사용자의 잠금 영역이 좌절이 아니라 업그레이드 동기로 보일 것
- **주의**: 추이 그래프(14일 바 차트, 오늘 강조)와 지출 삭제(탭→확인) 인터랙션 유지

### 🥉 3순위 — 장소·날씨 인사이트 (SPEC §5.11)
> **의뢰문**: SPEC §5.11 기준으로 인사이트 화면을 유료 핵심 상품답게 시각화해 주세요. 장소별 매출 랭킹, 날씨별 평균 매출(맑음/흐림/비/눈), 준비량 힌트("평균 N시간 만에 품절") 3개 섹션 + 각 섹션 빈 상태. 차트·시각 요소는 개발팀과 협의 가능한 수준의 단순한 형태(가로 바, 타일)로.
- **문제**: 현재 텍스트 리스트 수준 — "이 앱만의 데이터"라는 상품성이 안 보임
- **완료 기준**: 스크린샷 한 장으로 구독 가치가 전달될 것(스토어 스크린샷으로 재사용 예정)

### 4순위 — 신규 모달 3종 톤 통일 (SPEC §5.12, §5.13, §5.15)
> **의뢰문**: 영업 일정 캘린더, 지출 추가, 트럭 정보 편집 모달을 기존 menu-edit 모달과 일관된 패턴으로 정리해 주세요. 캘린더는 월 그리드(계획 점 표시·오늘 강조·과거 흐림)+날짜별 리스트+추가 폼 구조 유지.

### 5순위 — 예정 기능 선행 디자인 (SPEC §8.1)
> **의뢰문**: 다음 릴리스 예정 기능의 화면을 선행 디자인해 주세요: ① 전면광고 화면의 "광고 없이 쓰기 → 프로" 링크 배치(SPEC §5.4) ② 마감 리포트 푸시(알림 카드 + 탭 시 랜딩) ③ 주간 리포트 ④ 첫 실행 온보딩(메뉴 등록→장사 시작 가이드).

### 6순위 — 크로스커팅 요소 (SPEC §7)
> **의뢰문**: 전 화면 공통 요소를 컴포넌트화해 주세요: 빈 상태 일러스트 세트(메뉴 0/지출 0/인사이트 데이터 부족/일정 없음), 동기화 상태 pill·"지각 동기화" 배지, 체험중 배지, 직원 모드 헤더.

### 7순위 — 스토어 출시 그래픽
> **의뢰문**: Google Play 출시 자산 일체 — 앱 아이콘(적응형), 피처 그래픽(1024×500), 스크린샷 프레임 8장(핵심 플로우: 영업 시작→POS→마감→인사이트→페이월).

## D. 진행 방식 제안

- 1~3순위는 와이어프레임 단계에서 1회 중간 리뷰 후 하이파이 진행
- 4~6순위는 1~3순위에서 확정된 컴포넌트·톤을 적용해 일괄 진행
- 7순위(스토어 그래픽)는 3순위 인사이트 시안 확정 후 착수 (스크린샷 재사용)
