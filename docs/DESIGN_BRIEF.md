# 오늘장사 — 신규 디자인 의뢰서 (제로베이스)

작성일: 2026-07-10 · 발주: 오늘장사 개발팀
**전제: 기존 UI를 개선하는 작업이 아닙니다. 아래 기능 명세를 기준으로 IA(정보 구조), 내비게이션, 비주얼 아이덴티티, 전 화면을 처음부터 새로 설계해 주세요.** 현행 앱과 기존 시안(`design_handoff_foodtruck_pos/`)은 "이렇게 동작하는 물건이 이미 있다"는 참고 자료일 뿐, 따라야 할 대상이 아닙니다.

---

## 1. 제품 정의

- **한 줄**: 푸드트럭·노점 자영업자가 혼자서도 주문받고, 정산하고, "어디서 장사해야 돈이 되는지"까지 아는 앱.
- **플랫폼**: Android 스마트폰 (360~480px). 태블릿·PC 웹은 이번 범위 아님(단, 확장 가능성은 IA에 반영).
- **경쟁 대비 포지션**: 일반 POS(페이히어 등)는 매장용이다. 이 앱은 ① 인터넷 없어도 완전 동작, ② 매일 바뀌는 "장소"가 데이터의 축, ③ 사장 혼자 운영하는 손맛에 최적화.
- **수익모델**: 무료(광고) + 구독 "오늘장사 프로"(₩9,900/월·₩99,000/년).

## 2. 사용 맥락 — 디자인이 가장 크게 고려해야 할 것

| 상황 | 디자인 시사점 |
|---|---|
| **야외·직사광선** 아래에서 봄 | 고대비, 큰 숫자. 섬세한 회색조 의존 금지 |
| **서서, 한 손으로**, 조리 중간에 조작 | 엄지 도달 범위, 큰 터치 타깃, 오탭 복구 용이 |
| **주문 러시**: 대기줄 앞에서 초 단위 입력 | POS는 2탭 이내 주문 완료. 장식·애니메이션 최소 |
| 젖은 손, 위생장갑 | 스와이프·롱프레스 같은 정밀 제스처를 필수 동작으로 쓰지 말 것 |
| 마감 후 **정산은 차분한 순간** | 러시 화면(POS)과 정산·분석 화면의 밀도/톤 대비를 설계 자유 요소로 |
| 통신 불안정 (행사장·야외) | 오프라인이 기본 상태처럼 보여야 함. "연결 안 됨" 경고로 겁주지 말 것 |

## 3. 사용자와 상태 (모든 화면의 변형 축)

- **역할**: 사장(전 기능) / **직원**(주문 입력만 — 매출·정산·설정 접근 불가. 초대코드로 합류)
- **계정**: 게스트(로그인 없이 전 기능 로컬 사용) / 로그인(이메일·카카오 → 클라우드 동기화, 직원 초대)
- **요금제**: 무료(광고 노출, 분석 잠금) / **체험 24h**(보상형 광고 시청으로 획득, 분석만 해제·광고 유지) / **프로**(전 기능+광고 제거)

## 4. 핵심 사용자 여정 — "사장의 하루"

```
(전날) 일정 확인: 내일은 ○○공원 플리마켓
 ↓
출근 → 앱 열기 → 오늘 장소 확인(일정에서 자동 제안) → [장사 시작]
        └ 날씨 자동 기록됨 · 무료 사용자는 여기서 전면광고 1회
 ↓
주문 러시: 메뉴 탭탭 → 결제수단 선택 → 완료 (×80회 반복)
        └ 중간에: 품절 처리, 즉석 할인, 주문 취소, 자릿세 지출 입력
 ↓
[영업 종료] → 마감 정산: 오늘 매출·시재(카드/현금)·지출 차감 순이익
        └ 무료 사용자는 마감 후 전면광고 1회 · 8시간 넘게 안 닫으면 리마인더 알림
 ↓
(밤) 매출 확인 → "비 오는 날 ○○공원은 안 되는구나" (인사이트) → 다음 일정 등록
```

이 루프가 하루 단위로 반복된다. **"장사 시작~종료" 사이가 러시 모드, 그 밖은 관리 모드** — 두 모드의 구분이 IA의 핵심 과제.

## 5. 기능 요구사항

> 화면 구성은 자유. 단, 아래 기능·규칙·상태는 모두 커버해야 함.

### F1. 시작하기 (인증·온보딩)
- 로그인 없이 바로 사용 시작 가능해야 함(게스트) — 가입 강요는 이탈 1순위.
- 로그인 수단: 이메일+비밀번호, 카카오. 로그인하면: 동기화, 멀티 기기, 직원 초대 가능.
- 첫 진입 갈래: ① 내 트럭 만들기(상호+이름) ② 초대코드로 직원 합류.
- 첫 사용 가이드(메뉴 등록 → 장사 시작까지)를 새로 설계해 주세요 — 현행에는 없음.

### F2. 영업 세션
- 하루는 [장사 시작]으로 열리고 [영업 종료]로 닫힘. 세션이 열려 있어야 주문 입력 가능.
- 시작 시 입력/자동수집: **장소 태그**(선택 입력, 최근 5곳 재선택, 오늘 일정에서 자동 제안), **날씨**(위치 기반 자동, 실패 시 조용히 생략).
- 시작 화면에는 어제 실적 요약이 보임(동기부여).
- 시작 8시간 후에도 안 닫으면 로컬 알림 "마감 정산을 잊지 마세요".

### F3. 주문 입력 (POS) — 러시 모드의 전부
- 카테고리 → 메뉴 그리드 → 수량 → 결제수단(카드|현금|계좌이체|기타, 기본 카드) → 완료. **표준 주문은 2~3탭**.
- 부가 입력: 할인 메모, 총액 수동 조정(에누리).
- 메뉴 품절 처리(리스트에서 즉시 토글, 품절 시각은 자동 기록되어 인사이트 재료가 됨).
- 주문 취소: 삭제가 아니라 "취소됨" 표시로 남음(이력 보존).
- 화면 상단(또는 상시 영역): 영업 경과 시간, 동기화 상태.
- **이 화면에는 광고를 포함한 어떤 방해 요소도 금지.** 직원 계정은 이 기능만 사용 가능.

### F4. 메뉴 관리
- 메뉴 CRUD: 이름, 판매가, 원가, 카테고리, 품절.
- **레시피 원가 계산**: 재료(단가×수량)를 등록하면 원가 자동 산출. 원가율 건전성 표시(40% 이하 양호).
- 첫 사용자를 위한 "샘플 메뉴 채우기".

### F5. 매출 확인·분석
- 기간: 오늘(무료) / 이번 달·올해(**유료**).
- 보여줄 것: 매출·원가·순이익·주문수, **지출 차감 순이익**(진짜 오늘 번 돈 — 가장 중요한 숫자), 메뉴별 판매 랭킹, 주문 내역(시각·금액·결제수단·취소 표시)과 주문 상세.
- **추이 그래프**(유료): 최근 14일 일별 매출.
- 무료 사용자의 잠금 영역: 좌절감 대신 "광고 보고 24시간 체험" + "구독" 선택지를 제시 — 잠금 UX 자체가 전환 퍼널.

### F6. 지출 관리
- 기록: 금액 + 카테고리(자릿세|행사비|유류비|소모품|기타) + 메모. 러시 중에도 10초 안에 입력 가능해야 함.
- 기간별 지출 합계·목록, 삭제(취소 표시). 순이익 계산에 반영.
- 직원에게는 보이지 않음(사장 전용).

### F7. 인사이트 (유료 핵심 상품) — "이 앱만 주는 답"
- **장소별**: 어느 장소·행사가 세션당 평균 매출이 높은가 (랭킹).
- **날씨별**: 맑음/흐림/비/눈 별 평균 매출.
- **준비량**: "떡볶이는 평균 3시간 40분 만에 품절 — 더 준비하세요" (품절 기록 기반).
- 데이터가 쌓여야 가치가 나오는 기능 — 빈 상태("장소를 입력하고 장사하면 여기 쌓여요")가 기대감을 만들도록.
- 스토어 스크린샷의 주인공이 될 화면. 시각화 수준의 승부처.

### F8. 영업 일정
- 월 캘린더에 "몇 일, 어디서" 계획 등록(장소+메모). 계획 있는 날 표시, 다가오는 일정 목록.
- 오늘 일정은 장사 시작 시 장소로 자동 제안됨(F2 연계).

### F9. 직원(팀) 운영
- 사장이 초대코드 공유 → 직원 합류. 직원 목록(이름, PIN, 역할), 코드 재발급(기존 코드 무효화 경고).
- 직원 화면은 주문 입력 전용 — 매출 노출 없음.

### F10. 데이터·동기화
- **오프라인 우선**: 모든 조작은 즉시 저장되고, 온라인 시 자동 동기화. UI에는 "동기화 대기 N건" 정도의 조용한 표시와, 늦게 도착한 주문의 "지각 동기화" 배지.
- **CSV 내보내기**(세무사 전달용): 주문·지출. 무료는 최근 1개월, 유료 전체.

### F11. 수익화
- **전면광고**: 무료만, 세션 시작 전/마감 후만, 하루 3회 한도. 광고 로드 실패 시 그냥 통과(사용자는 실패를 모름). 광고 화면에 "광고 없이 쓰기 → 프로" 진입 노출.
- **네이티브 광고**: 무료만, 콘텐츠 리스트 속 카드형("광고" 라벨 필수), 실패 시 자리 소멸.
- **보상형 광고 = 체험권**: 잠금 기능에서 사용자가 자발적으로 "광고 보고 24시간 체험" 선택 → 끝까지 시청 시 해제. 체험 중 상태 표시(남은 시간).
- **페이월 "오늘장사 프로"**: 유료 기능 가치 제시, 연간(2개월 무료, 기본 선택)/월간, 실시간 스토어 가격, 구매 복원, Google Play 해지 고지. 상태: 결제 미설정/가격 로딩/로드 실패.
- 신규 가입 7일 광고 유예(예정) — 광고 등장 시점 커뮤니케이션 고려.

### F12. 설정·기타
- 트럭 정보(상호·사장님 이름) 편집, 요금제 상태·업그레이드, 로그인/로그아웃, 영업 종료.
- 마감 정산 화면: 오늘 매출·주문수·**결제수단별 시재**(카드 얼마, 현금 얼마)·지출·지출 차감 순이익.
- (예정) 마감 리포트 푸시("오늘 매출 ○○원, 지난주 대비 +12%"), 주간 리포트 — 알림 카드와 랜딩까지 선행 디자인 요청.

## 6. 불변 제약 — 어떤 IA를 제안하든 지켜야 함

1. 영업 중(세션 열림) 전면광고 0회. 주문 입력 화면엔 광고 절대 금지.
2. 광고·날씨·알림·결제 등 부가 기능은 **실패 시 조용히 사라짐** — 로딩 블로커·에러 모달로 영업을 막지 말 것.
3. 직원에게 매출·지출·설정 노출 금지.
4. 과거 주문은 불변(메뉴를 고쳐도 과거 기록은 그대로), 취소는 삭제가 아닌 표시.
5. 금액은 정수 원화(₩), 숫자 정렬은 고정폭. 한국어 전용, "~해요체", 장사 용어(시재·마감·품절) 유지.
6. Android 우선. 다크모드는 이번 범위에서 선택 사항(제안 환영).

## 7. 디자인 자유 범위

| 자유 (재설계 제안 환영) | 고정 |
|---|---|
| IA·내비게이션 구조 (탭 4개일 필요 없음) | §5의 기능 커버리지, §6 제약 |
| 비주얼 아이덴티티 전체 (색·서체·아이콘·일러스트) | 앱 이름 "오늘장사", 카카오 버튼 브랜드 규정 |
| 러시 모드/관리 모드의 구분 방식 | 무료/체험/프로 3단 요금제 구조 |
| 컴포넌트 체계 (기존 컴포넌트 무시 가능) | 가격 (₩9,900/월, ₩99,000/년) |
| 빈 상태·온보딩·전환 퍼널의 연출 | Google Play 정책 고지 문구류 |

참고: 현행 토큰(웜 오프화이트 `#F6F2EC`, 액센트 `#E85D3A`, Pretendard)은 "이랬다" 수준의 참고. 유지·폐기 모두 가능하나, 야외 시인성(§2)은 어떤 선택이든 검증할 것.

## 8. 산출물

1. **IA 제안** (화면 맵 + 내비게이션 구조 + 러시/관리 모드 해석) — 착수 후 1차 리뷰
2. **디자인 시스템**: 토큰(색·타이포·스페이싱), 핵심 컴포넌트, 라이트 기준(다크 제안 시 별도)
3. **전 화면 하이파이 + 인터랙티브 프로토타입**(Figma): §5 기능 전체, 화면당 상태 변형(무료/체험/프로 × 빈 상태 × 오프라인 × 역할) 포함
4. **개발 핸드오프 스펙**: 스페이싱·상태·모션 정의
5. **스토어 그래픽**: 앱 아이콘(적응형), 피처 그래픽(1024×500), 스크린샷 8장
