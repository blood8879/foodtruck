# 오늘장사 — 제품 로드맵 & 수익모델 설계

작성일: 2026-07-03
기준: 코드베이스 실사 결과 (M2 후반~M4 초반 상태)

---

## 1. 현재 상태 요약

**구현 완료 (실동작)**
- POS 주문 (장바구니, 할인메모, 총액조정, 원가 스냅샷)
- 메뉴 관리 (레시피 기반 원가 자동계산, 품절 토글)
- 매출 통계 (일 집계, 메뉴 TOP, 주문내역 / 월·연은 유료 게이팅)
- 영업 세션 (시작/마감), 직원 초대·역할 분리 (owner/staff)
- 로컬 우선 오프라인 + Supabase 이벤트소싱 동기화 (outbox/cursor)
- 전면 광고: 세션 시작/종료 시 무료 티어만, fail-open 설계

**미구현 (설계/seam만 존재하거나 부재)**
- 결제 (IAP) — entitlement 경계만 존재, `PlanTier` 로컬 토글 데모
- 추이 그래프 — 텍스트 플레이스홀더 (`sales.tsx`)
- 지출 관리, 재고, 영업일정, 날씨 연동 — 전무
- 데이터 내보내기, PC 웹 대시보드 — 설계만
- 초대코드 "발급" 버튼, 트럭 정보 편집 — 미연결
- 동기화 gap-free 프로덕션 검증 (M-GATE, `supabasePort.ts`)
- 결제수단(현금/카드/이체) 구분 — 주문 이벤트에 필드 없음

## 2. 사용자 관점 기능 갭 분석

푸드트럭/노점 자영업자의 실제 하루: **"오늘 어디서 장사하지" → 재료 준비 → 영업 → 마감 정산 → 세금/장부**.
현재 앱은 "영업" 구간만 커버한다. 나머지 구간이 기능 갭이자 차별화 기회다.

### 우선순위 P0 — 정산의 완결성 (없으면 "매출관리앱"이라 부르기 어려움)
| 기능 | 이유 |
|---|---|
| **결제수단 기록** (현금/카드/계좌이체/기타) | 마감 때 시재 정산의 기본. 현금매출 파악은 자영업자 최대 관심사 |
| **지출 관리** (자릿세, 행사참가비, 유류비, LPG, 소모품) | 현재 순이익 = 매출−재료원가뿐. 고정비를 빼야 "오늘 진짜 얼마 벌었나"가 나옴 |
| **추이 그래프 실구현** | 유료 게이팅의 핵심 상품인데 플레이스홀더 상태 |
| **CSV/엑셀 내보내기** | 세무사 전달·부가세 신고용. 자영업자 앱의 필수 탈출구이자 유료 기능 |

### 우선순위 P1 — 푸드트럭 킬러 기능 (일반 POS와의 차별화)
| 기능 | 이유 |
|---|---|
| **장소/행사 태그 + 장소별 매출 분석** | 푸드트럭은 매일 장소가 바뀜. "어느 자리·어느 행사가 돈이 되나"는 이 업종만의 질문이며 기존 POS가 못 푸는 문제 |
| **날씨 자동 기록 + 날씨×매출 상관** | 세션 시작 시 위치 기반 날씨를 이벤트에 자동 스탬프. "비 오는 날 ○○동 매출 −40%" 같은 인사이트 → 유료 분석의 재료 |
| **영업 일정 캘린더** | 행사 예약·출점 계획 관리. 장소 태그와 자연 결합 |
| **준비량 인사이트** | 품절 시각 기록 → "떡볶이는 평균 19시 품절, 20인분 더 준비하세요" |

### 우선순위 P2 — 편의/리텐션
- **마감 리포트 푸시**: 마감 후 "오늘 매출 ○○원, 지난주 같은 요일 대비 +12%" / 주간 요약 알림 → 재방문 습관 형성
- **카카오 로그인** (+ 넣는 순간 Apple 로그인 필수): 이메일 가입은 국내 자영업자에게 큰 이탈 지점
- **트럭 정보 편집, 초대코드 발급 버튼 연결** (기존 미연결 마감)
- 홈 위젯(오늘 매출), 다크모드는 후순위

### 장기 (검증 후)
- PC 웹 대시보드 (디자인 핸드오프에 5화면 설계 존재 — 유료 전용)
- 멀티 트럭/지점, 카드단말·배달앱 연동, 세무 리포트(부가세 추정)

## 3. 수익모델 설계

### 결론부터: "광고 1차 → 구독 2차"는 가능하다. 단, 역할을 정확히 나눠야 한다.

- **광고 = 무료 티어의 유지비 + 구독 전환 장치.** 니치 B2B 앱 특성상 광고 단독으로는 사업이 안 된다.
- **구독 = 본 수익.** 자영업자 도구는 "돈 버는 데 도움되면" 월 1만원 지불의사가 검증된 시장 (캐시노트, 페이히어 등).

#### 광고 수익 현실 추정 (한국 AdMob 기준)
| 형식 | eCPM 대략치 | 비고 |
|---|---|---|
| 전면 (Interstitial) | ₩15,000~40,000 | 현재 구현된 형식 |
| 보상형 (Rewarded) | ₩30,000~60,000 | 최고 단가, 거부감 최저 |
| 네이티브 | ₩5,000~15,000 | 리스트 삽입형 |
| 배너 | ₩1,000~3,000 | 단가 낮음, 오탭 리스크 |

- DAU 1,000명 가정: 전면 2회/일(시작·마감) = 월 6만 노출 ≈ **월 90만~240만원**
- 즉 광고 수익은 DAU에 선형 비례하며, 니치 앱 DAU 1,000은 이미 상당한 성공 지표. **광고는 서버비+α 수준으로 기대치를 잡는다.**

#### 구독 수익 추정
- 설계된 가격 ₩9,900/월 유지, **연간 ₩99,000 (2개월 무료)** 추가
- B2B 유틸리티 전환율 벤치마크 2~5% → MAU 10,000 × 3% = 월 ~297만원. 광고보다 상방이 크고 DAU 의존도가 낮음
- 구독 가치 패키지: 월/연 분석 + 추이 그래프 + **장소별/날씨 분석** + 지출 포함 순이익 리포트 + 데이터 내보내기 + **광고 제거** + (후속) PC 웹
- 구현: **RevenueCat** 권장 (스토어 IAP 영수증 검증·구독 상태 관리 대행, `entitlement.ts` seam에 그대로 연결됨)

## 4. 거부감 없는 광고 최대화 전략

### 대원칙: "영업 중에는 광고 0회"
활성 세션(장사 중) 동안 전면 광고가 뜨면 주문이 밀린다 → 즉시 삭제 사유. 현재 설계(세션 시작/종료 시만, fail-open 8초 타임아웃)가 이미 이 원칙을 지키고 있고, 이것이 이 앱 광고 전략의 뼈대다.

### 노출 지점 설계 (거부감 낮은 순 × 단가 높은 순)

1. **보상형 광고 = 유료 기능 1일 체험권** (최우선 추가)
   - "광고 보고 월간 분석 24시간 열어보기" — 사용자가 자발적으로 시청
   - 업계 최고 eCPM + **유료 기능을 직접 맛보게 하는 구독 전환 퍼널**을 겸함
   - 노출 위치: 매출 탭의 잠긴 월/연 토글, 추이 그래프 잠금 카드
2. **전면 광고 — 현행 유지 + 빈도 캡**
   - 세션 시작 전 / 마감 요약 후 (자연스러운 전환 순간, 이미 구현)
   - 하루 최대 2~3회 캡, 세션당 1회 유지, fail-open 유지
3. **네이티브 광고** (2차 추가)
   - 매출 탭 주문내역 리스트 N개당 1개, 마감 리포트 하단 1개
   - 배너보다 단가 높고 콘텐츠에 섞여 거부감 낮음
4. **배너는 최소화**: 넣는다면 설정 화면 하단 앵커드 어댑티브 배너만. **POS 화면 절대 금지** (오탭 = 주문 실수)
5. **앱 오픈 광고 금지**: 급하게 주문 받으려 앱을 여는 POS 특성상 최악의 형식

### 거부감 완화 장치
- **신규 가입 후 7일 광고 유예** — 온보딩 기간 리텐션 보호, "광고가 늘었네" 대신 "원래 이런 앱"으로 인지
- 광고 화면에 항상 "광고 없이 쓰기 → 구독" 링크 노출 (거부감을 전환으로 회수)
- 로드 실패 시 절대 대기시키지 않는 fail-open 유지
- 필수 준수: **UMP 동의(GDPR/GAD), iOS ATT 프롬프트** — 미처리 시 노필/정책 위반
- 미디에이션(AppLovin 등 추가)은 월 100만 노출 규모 이후에 검토. 초기엔 AdMob 단독으로 충분

## 5. 단계별 로드맵

### Phase 0 — 출시 전 마감 (2~4주)
> 목표: 스토어에 낼 수 있는 완성도
- [x] 추이 그래프 실구현 (일/주 매출 차트)
- [x] 결제수단 필드 추가 (`OrderPlaced` 이벤트 확장 + 마감 시재 요약)
- [x] 미연결 UI 마감: 초대코드 발급, 트럭 정보 편집
- [x] 동기화 gap-free 검증 (M-GATE 해소) 및 장애 시나리오 테스트
- [x] UMP/ATT 동의 플로우, 광고 하루 빈도 캡

### Phase 1 — 출시 + 광고 수익화 (1~2개월)
> 목표: 무료 사용자 확보, 광고 파이프라인 완성
- [x] 지출 관리 (카테고리: 자릿세/행사비/유류비/기타 → 순이익 반영)
- [x] 장소/행사 태그 (세션 시작 시 입력, 세션 이벤트에 스탬프)
- [x] CSV 내보내기 (무료: 최근 1개월 / 유료: 전체)
- [x] 보상형 광고 "유료 기능 1일 체험" 도입
- [x] 마감 리마인더 푸시 (세션 +8시간, 마감 시 취소 — 리포트 푸시는 서버 푸시 필요로 Phase 2+)
- [ ] 스토어 출시 (iOS/Android)

### Phase 2 — 리텐션 & 차별화 (2~3개월)
> 목표: "푸드트럭엔 이 앱뿐" 포지셔닝, 구독 가치 축적
- [x] 날씨 자동 기록 + 날씨×장소×매출 분석 (유료 핵심 상품)
- [x] 영업 일정 캘린더 (+ 오늘 일정 → 영업 시작 장소 자동 채움)
- [x] 준비량 인사이트 (품절 시각 기반)
- [x] 네이티브 광고 (매출 리스트/마감 리포트, 무료 티어만)
- [x] 카카오 로그인 (Apple 로그인은 iOS 미지원 방침으로 제외)

### Phase 3 — 구독 전환 (1~2개월)
> 목표: 2차 수익모델 가동
- [ ] RevenueCat + 스토어 IAP 연동 (`entitlement.ts` seam에 연결)
- [ ] 페이월 화면 (₩9,900/월, ₩99,000/년, 7일 무료 체험)
- [ ] 광고 제거를 구독에 포함
- [ ] 전환 계측: 보상형 체험 → 구독 퍼널 추적

### Phase 4 — 확장 (검증 후)
- [ ] PC 웹 대시보드 (설계 완료분, 유료 전용)
- [ ] 멀티 트럭/지점
- [ ] 세무 리포트 (부가세 추정), 카드단말/배달앱 연동 검토

## 6. 성공 지표 (Phase별 게이트)

| Phase | 게이트 지표 |
|---|---|
| 1 → 2 | D30 리텐션 ≥ 20%, 주간 세션(영업일) ≥ 3회 |
| 2 → 3 | 보상형 체험 사용률 ≥ 15% (구독 수요 신호) |
| 3 평가 | 무료→유료 전환율 ≥ 2%, 광고 ARPDAU ≥ ₩30 |
