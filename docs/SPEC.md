# 오늘장사 — 기능 명세서 (디자인 의뢰용)

작성일: 2026-07-08 · 기준 코드: main `549f759`
용도: 디자인팀에 와이어프레임·세부 UI 개선을 의뢰하기 위한 기능/화면 명세.
기존 하이파이 시안: `design_handoff_foodtruck_pos/` (13개 화면 PNG + HTML 레퍼런스) — 본 문서의 "구현 완료" 화면 다수가 이 시안을 따르고 있으며, 시안이 없는 신규 화면(§5.11~5.15)이 이번 의뢰의 핵심입니다.

---

## 1. 제품 개요

- **한 줄 정의**: 푸드트럭·노점 자영업자를 위한 오프라인 우선(local-first) POS + 매출·지출·인사이트 관리 앱.
- **플랫폼**: Android 전용 모바일 앱 (Expo React Native). PC 웹 대시보드는 설계만 존재(§8.3).
- **차별점**: ① 인터넷 없이 완전 동작(동기화는 온라인 시 자동), ② 장소가 매일 바뀌는 업종 특화 — 장소·날씨별 매출 분석, ③ 영업 흐름(시작→주문→마감)을 방해하지 않는 광고 설계.
- **수익모델**: 1차 광고(무료 티어), 2차 구독(₩9,900/월, ₩99,000/년 — "오늘장사 프로").

## 2. 사용자 유형과 권한 매트릭스

| 축 | 값 | 의미 |
|---|---|---|
| 역할 | **사장(owner)** | 모든 탭·기능 접근. 트럭 생성자 |
| | **직원(staff)** | **주문 탭만** 노출(매출/메뉴/설정 숨김). 초대코드로 합류 |
| 계정 | **게스트** | 로그인 없이 전 기능 로컬 사용(동기화·직원 합류 불가) |
| | **로그인** | 이메일 또는 카카오. 클라우드 동기화·멀티 기기·직원 초대 |
| 요금제 | **무료** | POS·메뉴·일 매출·지출·일정 전부 사용 가능. 광고 노출. 유료 분석 잠금 |
| | **체험(24h)** | 보상형 광고 시청으로 획득. 유료 분석·내보내기 해제. **광고는 계속 노출** |
| | **프로(구독)** | 전 기능 + 광고 제거 |

유료 잠금 기능: 기간 분석(월/연), 추이 그래프, 장소·날씨 인사이트, 데이터 내보내기(전체 기간), PC 웹(예정), 광고 제거(구독 전용, 체험 제외).

## 3. 도메인 개념 (디자인에 영향을 주는 것만)

- **영업 세션**: "장사 시작"으로 열고 "영업 종료"로 닫는 하루의 단위. 세션이 없으면 POS 진입 불가(영업 시작 화면으로 리다이렉트). 세션에는 장소 태그·날씨가 박제됨.
- **주문**: 라인(메뉴×수량)의 판매가·원가가 판매 시점에 스냅샷됨. 이후 메뉴를 수정해도 과거 주문 불변. 취소(void)는 삭제가 아니라 취소 표시.
- **순이익 2단계**: `순이익(마진) = 매출 − 재료원가`, `지출 차감 순이익 = 순이익 − 지출(자릿세·유류비 등)`. 화면에서 두 값이 구분 표기됨.
- **오프라인 우선**: 모든 조작은 즉시 로컬 저장. 온라인이 되면 자동 동기화. 디자인 요소: 상단 동기화 상태 pill("동기화 대기 N건"), 늦게 도착한 주문의 "지각 동기화" 배지.
- **결제수단**: 카드/현금/계좌이체/기타 — 주문 시 선택, 마감 시재 요약의 축.

## 4. 정보 구조 (화면 맵)

```
로그인(sign-in) ── 게스트로 둘러보기 ─┐
   └─ 온보딩(트럭 생성/초대코드 합류) ─┤
                                      ▼
                          영업 시작(session-start)  ←──[영업 종료 후 복귀]
                                      │ 장사 시작 (무료: 전면광고 경유 /ad)
                                      ▼
        ┌─────────── 하단 탭 (사장 4개 / 직원 1개) ───────────┐
        │  주문(POS)   매출        메뉴        설정            │
        └──┬───────────┬───────────┬───────────┬──────────────┘
           │           │           │           ├─ 트럭 정보 편집 (모달)
           │           │           │           ├─ 영업 일정 캘린더 (모달)
           │           │           │           ├─ 데이터 내보내기 (공유 시트)
           │           │           │           ├─ 페이월 (모달)
           │           │           └─ 메뉴 추가/수정 (모달)
           │           ├─ 주문 상세 (모달)
           │           ├─ 지출 추가 (모달)
           │           ├─ 인사이트 (모달, 유료)
           │           └─ 페이월 (모달)
           └─ [설정→영업 종료] → 마감 요약(close-summary) → (무료: 전면광고) → 영업 시작
```

## 5. 화면별 상세 명세 — 구현 완료

### 5.1 로그인 (sign-in)
- 요소: 이메일/비밀번호 폼, **카카오로 시작하기**(#FEE500 브랜드 버튼, "또는" 구분선), 회원가입 전환, **"로그인 없이 둘러보기"**(게스트).
- 분기: Supabase 미설정 빌드에서는 카카오·이메일 숨김(게스트만).
- 상태: 로그인 실패 에러 문구 인라인.

### 5.2 온보딩 (onboarding)
- 로그인했지만 소속 트럭이 없을 때만 진입.
- 두 갈래: **트럭 만들기**(상호+사장님 이름 → owner) / **초대코드로 합류**(코드+이름 → staff).

### 5.3 영업 시작 (session-start) — 앱의 홈
- 요소: 오늘 날짜 + **날씨 배지**("☁️ 12°C 흐림", 백그라운드 자동 수집·없으면 미표시), 어제 매출/순이익 요약 카드, **오늘 장사 장소 입력**(선택) + 최근 장소 칩 5개, **오늘 일정이 있으면 장소 자동 채움**, "장사 시작" CTA.
- 동작: 시작 시 무료 티어는 전면광고 화면(/ad) 경유 → 주문 탭. 마감 리마인더(+8h 로컬 알림) 예약.
- 상태: 게스트/로그인 공통. 어제 매출 0이면 요약 카드 0원 표시.

### 5.4 전면광고 (/ad)
- 무료 티어만, 세션 시작/종료 시. 하루 최대 3회.
- **fail-open**: 로드 실패·8초 초과 시 광고 없이 즉시 통과. 광고가 영업을 막는 일은 없음.
- 개선 여지(디자인 의뢰): 광고 대기 화면에 "광고 없이 쓰기 → 프로 구독" 링크(§8.1).

### 5.5 주문 POS (탭 1 — 유일하게 직원도 접근)
- 상단: 영업시간 경과 타이머, 동기화 상태 pill.
- 본문: 카테고리 칩 → 메뉴 그리드(품절 표시), 장바구니 시트(수량 스테퍼, 할인 메모, 총액 수동조정).
- 결제: **결제수단 세그먼트(카드|현금|이체|기타, 기본 카드)** → "주문 완료 · ₩합계" 버튼. 완료 후 장바구니·결제수단 리셋.
- 엣지: 중복 탭 방지, 품절 메뉴 선택 불가.

### 5.6 매출 (탭 2) — 정보 밀도가 가장 높음, 개선 최우선
현재 세로 스크롤 구성(위→아래):
1. 타이틀 + **체험중 배지**("체험중 · N시간 남음")
2. 기간 토글: 오늘 | 이번 달(유료) | 올해(유료) — 잠금 시 자물쇠 + **"광고 보고 24시간 체험" 버튼** + "구독하기" 링크
3. KPI 카드: 매출/원가/순이익/주문수 + **지출 차감 순이익** 카드(지출 있을 때)
4. **추이 그래프**(유료): 최근 14일 바 차트, 오늘 강조, 합계·일평균
5. **장소·날씨 인사이트 진입 카드**(유료, 잠금 시 체험 버튼)
6. 메뉴별 판매 TOP 랭킹
7. **지출 섹션**: 기간 지출 합계, 항목 리스트(카테고리 배지+메모+금액, 탭=삭제 확인), "+ 지출 추가"
8. 주문 내역 리스트(시각·금액·결제수단, 취소건 표시) — **5번째 뒤 네이티브 광고 1개(무료만)**
- 디자인 과제: 섹션이 8개로 과밀. 요약/상세의 위계 재설계 필요.

### 5.7 메뉴 관리 (탭 3)
- 카테고리별 그룹 리스트: 메뉴명, 판매가, **원가율 칩**(40% 이하 녹색), 품절 토글.
- 빈 상태: "샘플 메뉴 채우기" 버튼.
- 메뉴 추가/수정 모달: 이름/판매가/원가/카테고리/품절 + **레시피 편집**(재료 단가×수량 합산 → 원가 자동 계산).

### 5.8 설정 (탭 4)
1. 트럭 정보 카드(상호/사장님) → 편집 모달
2. **영업 일정** 진입 → 캘린더 모달
3. 직원 관리: 직원 리스트(아바타·이름·PIN·역할 배지), 초대코드 표시 + **재발급 버튼**(확인 다이얼로그: 기존 코드 무효화)
4. **데이터 내보내기**: 주문·지출 CSV → OS 공유 시트. 무료는 최근 1개월(안내 문구), 유료/체험 전체
5. 요금제 카드(다크): 유료 기능 리스트, **"프로 업그레이드" → 페이월**, 체험중 표시
6. 영업 종료 버튼 / 로그인·로그아웃

### 5.9 마감 요약 (close-summary)
- 오늘 매출/순이익/주문수 + **오늘 지출·지출 차감 순이익** + **시재 요약**(결제수단별 금액, 0원 수단 숨김).
- 하단: 네이티브 광고 1개(무료만) → "영업 종료" CTA → (무료: 전면광고) → 영업 시작 화면.
- 마감 시 리마인더 알림 취소.

### 5.10 주문 상세 (모달)
- 라인 내역(스냅샷 가격), 할인 메모, 조정총액, 순이익, **결제수단**, 주문 취소(void) 버튼(사유 없이 1탭 + 확인).

### 5.11 인사이트 (모달, 유료) — 신규, 시안 없음 ★
- 섹션 3개: **장소별 매출**(장소·세션 N회·평균/총매출 랭킹), **날씨별 평균 매출**(맑음/흐림/비/눈 + 이모지), **준비량 힌트**("떡볶이, 평균 3시간 40분 만에 품절 — 준비량을 늘려보세요", 2회 이상 관측만).
- 각 섹션 빈 상태 문구 존재("장소 태그를 입력하고 장사하면 여기에 쌓여요").
- 디자인 과제: 현재 텍스트 리스트 수준. 유료 핵심 상품답게 시각화(차트·지도 뉘앙스) 필요.

### 5.12 영업 일정 캘린더 (모달) — 신규, 시안 없음 ★
- 월 그리드(일~토, 오늘 강조, 계획 있는 날 점 표시, 과거 흐림), ◀▶ 월 이동.
- 날짜 탭 → 하단에 해당일 계획 리스트(장소·메모, 탭=삭제) + 추가 폼(장소/메모).
- "다가오는 일정" 5개 리스트. 오늘 일정의 장소는 영업 시작 화면에 자동 반영됨(§5.3).

### 5.13 지출 추가 (모달) — 신규, 시안 없음 ★
- 금액(숫자), 카테고리 칩(자릿세|행사비|유류비|소모품|기타, 기본 자릿세), 메모(선택), 저장.

### 5.14 페이월 (모달) — 신규, 시안 없음 ★★
- "오늘장사 프로" 헤더, 유료 기능 체크리스트, 상품 카드 2개(연간 기본 선택+"2개월 무료" 배지 / 월간), 가격은 스토어 실시간 표시, "구독하기" CTA, "구매 복원" 텍스트 버튼, Google Play 해지 고지.
- 상태 3종: 결제 미설정("결제 준비 중이에요"), 가격 로딩, 상품 로드 실패.
- 디자인 과제: 전환율을 좌우하는 화면. 가치 전달·비교·신뢰 요소 전면 재설계 의뢰.

### 5.15 트럭 정보 편집 (모달)
- 상호(필수)/사장님 이름, 저장. 단순 폼.

## 6. 수익화 UX 규칙 (디자인 제약 조건 — 위반 불가)

1. **영업 중(활성 세션) 전면광고 0회.** 전면은 세션 시작 전/마감 후만, 하루 3회 캡.
2. **POS 화면에는 어떤 광고도 금지** (오탭 = 주문 실수).
3. 네이티브 광고는 콘텐츠 카드 톤 + **"광고" 라벨 필수**, 로드 실패 시 자리 자체가 사라짐(스켈레톤 금지).
4. 보상형 광고는 항상 **사용자 자발 선택**(체험 버튼) — 강제 노출 없음. 끝까지 시청해야 보상.
5. 광고·결제·날씨·알림 등 모든 부가 기능은 실패 시 조용히 사라질 뿐, **영업 흐름을 절대 막지 않음**(fail-open) — 로딩 블로커 디자인 금지.
6. 체험(24h)은 분석 기능만 해제, 광고 제거는 구독 전용.

## 7. 크로스커팅 상태 (모든 화면 공통으로 디자인 필요)

- **오프라인/동기화**: 동기화 대기 pill, 지각 동기화 배지, 게스트(동기화 없음) 상태.
- **권한 거부**: 위치(날씨 미표시), 알림(리마인더 무음 생략) — 별도 에러 UI 없이 기능만 조용히 빠짐.
- **빈 상태**: 메뉴 0개(샘플 채우기), 지출 0건, 인사이트 데이터 부족, 일정 없음 — 각각 문구 존재, 일러스트 등 개선 여지.
- **역할 분기**: 직원은 탭 1개 — 직원용 화면의 단순화된 헤더/네비 고려.
