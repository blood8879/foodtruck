# 오늘장사 — 화면·기능 명세서 (외부 디자인팀 의뢰용)

작성일: 2026-07-11 · 기준 코드: main 최신 (전 기능 실코드 확인 후 작성)
**본 문서 하나로 와이어프레임·세부 UI 작업을 질문 없이 시작할 수 있도록 작성했습니다.**
전제: **제로베이스 신규 디자인** — 기존 앱의 IA·데이터 구조는 확정이지만, 비주얼·레이아웃·내비게이션 형태는 전면 재량입니다. 구버전 시안(`design_handoff_foodtruck_pos/`)은 참고용이며 비구속입니다.

## 0. 제품 컨텍스트

- **앱**: 오늘장사 — 푸드트럭·노점 자영업자가 혼자서 주문받고, 정산하고, "어디서 장사해야 돈이 되는지"까지 아는 오프라인 우선 장사 앱.
- **타겟 유저**: 1인~2인 운영 푸드트럭/노점 사장. 야외에서, 서서, 한 손으로, 주문 러시 중에 씀. 보조 사용자로 "직원"(주문 입력만 하는 알바).
- **시각적 주인공**: **돈 숫자.** 화면마다 주인공이 되는 금액이 하나씩 있다 — POS에서는 "주문 합계", 마감·매출에서는 "지출 차감 순이익"(오늘 진짜 번 돈), 페이월에서는 "₩9,900/월". 모든 화면에서 해당 금액이 가장 크고 가장 먼저 읽혀야 한다. 금액은 항상 고정폭 숫자(tabular numerals).
- **플랫폼·제약**: Android 모바일 전용, 기준 뷰포트 360~480px. 라이트 모드 기준(다크는 §5 참조). 한국어 전용, "~해요체". 야외 직사광선 시인성 필수.
- **수익 모델**: 무료(광고 3종) + 구독 "오늘장사 프로" ₩9,900/월·₩99,000/년 → **페이월 화면 필요**, 광고 슬롯 디자인 필요, "보상형 광고=24시간 체험권" 퍼널 디자인 필요.

### 3.2 법적·정책 필수 문구 (원문 유지)
- 페이월 하단: **"구독은 Google Play 계정으로 청구되며, 언제든지 Google Play에서 해지할 수 있어요. 해지 후에도 남은 기간까지 프로 기능이 유지됩니다."**
- 페이월에 "구매 복원" 진입 필수(Google 정책).
- 모든 네이티브 광고 카드에 **"광고"** 라벨 필수 — 콘텐츠로 위장 금지(AdMob 정책).
- 위치 권한 요청 맥락: 날씨 자동 기록 용도임이 드러나야 함.

### 3.3 금지 사항
- 수익 보장형 문구 금지("무조건 돈 법니다" 류) — 인사이트는 "참고"의 톤.
- 체험권을 "무료"로만 표기 금지 — "24시간"이 항상 병기.
- 광고 로드 실패 시 빈 자리·스켈레톤·에러 표시 금지 — **자리 자체가 사라지는 것으로 디자인**.
- 영업 중(세션 열림) 전면광고, 주문(POS) 화면 내 모든 광고 금지.
- 필수 동작에 스와이프·롱프레스 의존 금지(젖은 손·장갑 사용 환경).

### 3.4 전 화면 공통 상태 (각각 디자인 필요)
1. **빈 상태**: 메뉴 0개("샘플 메뉴 채우기" 액션 포함) / 오늘 주문 0건 / 지출 0건 / 인사이트 데이터 부족("장소를 입력하고 장사하면 여기 쌓여요") / 일정 없음 / 직원 없음
2. **오프라인·동기화**: 동기화 대기 pill(조용한 상시 표시 — 경고 아님), 지각 동기화 배지
3. **로딩**: 원칙적으로 로컬 데이터라 로딩이 거의 없음. 필요한 곳은 페이월 가격 로딩("가격 불러오는 중"), 날씨(로드 전엔 아예 미표시). **전면 로딩 스피너·블로킹 오버레이 금지**
4. **오류**: 페이월 상품 로드 실패 / CSV 내보내기 실패 — 인라인 안내+재시도. 그 외 부가 기능(날씨·광고·알림)은 실패를 사용자에게 보이지 않음
5. **권한 거부**: 위치(날씨 없이 진행) / 알림(리마인더 없이 진행) — 별도 화면 불필요, 기능만 조용히 생략

## 4. 화면별 명세

> 형식 — **목적**: 이 화면이 3초 안에 답할 질문 / **구성**: 위→아래 / **데이터 규칙** / **인터랙션** / **예정 추가** / **유의점**

### S1. 로그인 ✅
- **목적**: "가입 안 해도 쓸 수 있나?" → 예.
- **구성**: 앱 아이덴티티 → 카카오로 시작하기(브랜드 #FEE500, 검정 텍스트) → "또는" → 이메일/비밀번호+가입 전환 → **로그인 없이 둘러보기**(게스트).
- **데이터 규칙**: 로그인 실패 에러는 인라인 문구.
- **인터랙션**: 카카오=외부 브라우저 왕복(취소 시 무반응 복귀).
- **예정 추가**: 없음.
- **유의점**: 게스트 진입을 죄책감 없이 크게. 게스트여도 전 기능 사용 가능(동기화·직원만 불가)임을 카피로.

### S2. 온보딩(트럭 설정) ✅
- **목적**: "나는 사장인가 직원인가?"
- **구성**: 두 갈래 카드 — ① 내 트럭 만들기(상호+사장님 이름) ② 초대코드로 합류(코드 5자리+내 이름).
- **데이터 규칙**: 초대코드는 영문대문자+숫자 5자리(`1D4EB`).
- **예정 추가 🔜**: 트럭 생성 직후 "첫 메뉴 등록 → 장사 시작" 가이드 플로우(신규 디자인).

### S3. 영업 시작 ✅ — 관리 모드의 홈
- **목적**: "오늘 장사 시작할 준비 됐나? 어제는 얼마 벌었나?"
- **구성**: 오늘 날짜+날씨 배지(있을 때만) → 어제 실적 카드(매출·순이익) → 오늘 장소 입력(선택)+최근 장소 칩 최대 5개 → [장사 시작] CTA.
- **데이터 규칙**: 오늘 일정에 장소가 있으면 입력란 자동 채움(수정 가능). 날씨는 §3.1 형식.
- **인터랙션**: 시작 → (무료) 전면광고 경유 → 주문 화면. 위치 권한 최초 1회 요청.
- **예정 추가 🔜**: 오늘 일정 미리보기 카드.
- **유의점**: 하루의 시작 의식(ritual). 어제 실적이 동기부여가 되도록 — 단 0원인 날 초라하지 않게.

### S4. 전면광고 게이트 ✅
- **목적**: (무료 사용자) 광고 1회 시청 → 통과.
- **구성**: 광고 SDK 전면 노출. 웹/실패 시 폴백: 짧은 카운트다운 화면.
- **인터랙션**: 로드 실패·8초 초과 시 자동 통과(사용자는 실패를 모름). 하루 3회 초과 시 이 화면 자체를 건너뜀.
- **예정 추가 🔜**: 폴백/직후 화면에 "광고 없이 쓰기 → 프로" 진입.
- **유의점**: 시작 러시 직전이므로 어떤 추가 탭도 요구하지 말 것.

### S5. 주문 POS ✅ — 러시 모드, 앱의 척추. **직원은 이 화면만 사용**
- **목적**: "지금 이 손님 주문을 몇 초 안에 넣을 수 있나?"
- **구성**: 상단 얇은 바(영업 경과 시간 `2:31:07`+동기화 pill) → 카테고리 칩 → 메뉴 그리드(이름·가격·품절 상태) → 장바구니 영역(라인·수량 스테퍼·할인 메모·총액 조정) → 결제수단 세그먼트(카드|현금|이체|기타, 기본 카드) → [주문 완료 · ₩13,000].
- **데이터 규칙**: 주인공 금액=주문 합계. 품절 메뉴는 선택 불가가 명확히.
- **인터랙션**: 표준 주문 2~3탭. 완료 시 장바구니·결제수단 리셋(기본 카드). 연타 방지.
- **예정 추가**: 없음.
- **유의점**: **광고·배너·팁 등 일체 금지.** 햇빛 아래 명도 대비. 직원 모드에선 탭바 없이 이 화면이 전부 — 단독으로도 성립하는 레이아웃.

### S6. 매출 ✅ — 관리 모드의 중심, 정보 위계 재설계 요망
- **목적**: "오늘(이번 달) 진짜 얼마 벌었나?"
- **구성(현행 순서, 재배열 재량)**: 기간 토글(오늘|이번 달 🔒|올해 🔒) → KPI(매출·원가·순이익·주문수) → **지출 차감 순이익**(주인공) → 14일 추이 바 차트 🔒(오늘 강조·합계·일평균) → 인사이트 진입 카드 🔒 → 메뉴 TOP 랭킹 → 지출 섹션(합계·목록·추가 버튼) → 주문 내역(**5번째 뒤 네이티브 광고 1개, 무료만**).
- **데이터 규칙**: 🔒=유료 잠금 — 잠금 영역에는 [광고 보고 24시간 체험]+[구독하기] 두 선택지. 체험중이면 헤더에 "체험중 · N시간 남음" 배지. 취소 주문은 취소선+배지.
- **인터랙션**: 지출 항목 탭→삭제 확인. 주문 탭→주문 상세. 체험 버튼→보상형 광고(§2 퍼널).
- **예정 추가 🔜**: 주간 리포트 진입.
- **유의점**: 현행 8개 섹션 세로 나열로 과밀 — 요약→상세 위계 재설계가 이 화면의 과제. 잠금이 "좌절"이 아니라 "미리보기+열쇠 2개"로 보이게.

### S7. 주문 상세 ✅ (모달)
- **목적**: "이 주문 뭐였고, 취소해야 하나?"
- **구성**: 시각·주문 라인(메뉴×수량×단가)·할인 메모·조정총액(있으면)·순이익·결제수단 → [주문 취소].
- **데이터 규칙**: 가격은 판매 시점 스냅샷(현재 메뉴 가격과 다를 수 있음 — 표기 그대로).
- **인터랙션**: 취소는 확인 다이얼로그 1회. 취소된 주문은 버튼 대신 취소 상태 표시.

### S8. 지출 추가 ✅ (모달)
- **목적**: "자릿세 5만원 냈다 — 10초 안에 기록."
- **구성**: 금액(숫자 키패드) → 카테고리 칩 5개(기본 자릿세) → 메모(선택) → [저장].
- **유의점**: 러시 중에도 쓰는 화면. 금액 먼저, 큰 입력.

### S9. 메뉴 관리 ✅
- **목적**: "뭘 팔고 있고, 마진은 괜찮나?"
- **구성**: 카테고리 그룹 리스트 — 메뉴명·판매가·원가율 칩(§3.1 색 규칙)·품절 토글 → [+ 추가]. 빈 상태: 샘플 메뉴 채우기.
- **인터랙션**: 품절 토글 즉시 반영(POS 연동, 시각 자동 기록 — UI에 노출 안 함).

### S10. 메뉴 추가/수정 ✅ (모달)
- **목적**: "메뉴 하나 등록."
- **구성**: 이름·판매가·카테고리·품절 → 원가 섹션: 직접 입력 **또는** 레시피(재료명·단가·단위·수량 행 추가 → 원가 자동 합산·원가율 미리보기) → 저장.
- **유의점**: 레시피는 파워유저 기능 — 접히거나 단계적 노출 재량.

### S11. 인사이트 ✅ (유료) — 스토어 스크린샷의 주인공
- **목적**: "어디서, 어떤 날씨에 장사해야 돈이 되나?"
- **구성**: ① 장소별 매출(장소·세션 N회·평균/총매출 랭킹) ② 날씨별 평균 매출(맑음/흐림/비/눈) ③ 준비량 힌트("떡볶이, 평균 3시간 40분 만에 품절 — 준비량을 늘려보세요", 2회 이상 관측만 표시).
- **데이터 규칙**: 장소 미입력 세션은 "미지정"으로 집계. 각 섹션 독립적 빈 상태(§3.4).
- **예정 추가**: 없음(데이터 축적으로 자연 성장).
- **유의점**: 유료의 얼굴. 텍스트 나열이 아니라 시각화(가로 바·타일 수준의 구현 가능한 형태)로. 한 화면 캡처로 구독 가치가 전달될 것.

### S12. 영업 일정 ✅ (모달)
- **목적**: "다음 주 어디서 장사하지?"
- **구성**: 월 헤더(◀ 2026년 7월 ▶) → 요일 그리드(오늘 강조·계획 점·과거 흐림) → 선택일 계획 목록(장소·메모, 탭=삭제) → 추가 폼(장소·메모) → 다가오는 일정 5개.
- **데이터 규칙**: 오늘 일정의 장소가 영업 시작 화면에 자동 제안됨 — 연결감을 카피로 알려도 좋음.

### S13. 설정 ✅
- **목적**: "장사 외 관리 전부."
- **구성**: 트럭 정보 카드(→편집 모달 S14) → 영업 일정 진입 → 직원 관리(목록: 이름·PIN·역할 배지 / 초대코드 표시+[재발급]) → 데이터 내보내기(주문·지출 CSV→공유 시트, 무료는 "최근 1개월만" 안내) → 요금제 카드(현재 상태·체험 잔여·[프로 업그레이드]) → 영업 종료 → 로그인/로그아웃.
- **인터랙션**: 재발급=확인 다이얼로그("기존 초대코드는 즉시 무효화됩니다").
- **유의점**: 영업 종료가 설정에 있는 것은 현행일 뿐 — 러시 모드에서 마감으로 가는 더 좋은 동선 제안 환영.

### S14. 트럭 정보 편집 ✅ (모달): 상호(필수)·사장님 이름 → 저장. 단순 폼.

### S15. 마감 정산 ✅ — 하루의 마침표
- **목적**: "오늘 진짜 얼마 벌었고, 시재는 맞나?"
- **구성**: 완료 아이콘+헤드라인 → 오늘 매출·주문수 → **시재 요약**(결제수단별 금액, 0원 수단 숨김) → 오늘 지출·**지출 차감 순이익**(주인공) → 네이티브 광고 1개(무료만) → [영업 종료 확인].
- **인터랙션**: 종료 → (무료) 전면광고 → 영업 시작 화면 복귀.
- **예정 추가 🔜**: 마감 리포트 푸시 프리뷰("이 내용을 밤 9시에 알림으로 받아보세요").
- **유의점**: 차분한 순간 — 러시 화면과 톤 대비. 하루 노고의 보상이 되는 연출 여지.

### S16. 페이월 "오늘장사 프로" ✅ — 전환율 결정 화면
- **목적**: "₩9,900으로 뭘 얻는가?"
- **구성**: 프로 아이덴티티 헤더+닫기 → 가치 제안 한 줄 → 유료 기능 체크리스트(기간 분석/추이 그래프/장소·날씨 인사이트/데이터 내보내기/광고 제거/PC 웹(예정)) → 상품 카드: 연간 ₩99,000(기본 선택, "2개월 무료" 배지, 월 환산 표기 재량)·월간 ₩9,900 → [구독하기] → 구매 복원(텍스트) → §3.2 고지 문구.
- **데이터 규칙**: 가격은 스토어 실시간 값(디자인은 자릿수 여유 확보). 상태 3종: 결제 미설정("결제 준비 중이에요")/가격 로딩/상품 로드 실패(재시도).
- **인터랙션**: 상품 카드 선택(라디오) → 구매. 취소는 무반응, 실패만 알림.
- **유의점**: 체험 만료 직전 유입이 많음 — "체험으로 맛본 것을 잃지 않기" 프레임 활용 가능.

### S17. 🔜 마감 리포트 푸시 + 랜딩 (신규 와이어프레임)
- 알림: "오늘 매출 ₩487,000 · 지난주 금요일보다 +12%" → 탭 시 그날 마감 요약 랜딩. 주간 리포트(주 1회 요약)도 동일 계열로.

## 5. 디자인 시스템 요청사항

- **브랜드**: 로고·앱 아이콘·컬러 시스템 **신규 제안 필요**(현행은 개발자 임시: 웜 오프화이트 `#F6F2EC`+오렌지레드 `#E85D3A`+Pretendard — 유지·폐기 재량, 야외 시인성만 검증). 앱 이름 "오늘장사"는 고정.
- **최소 컴포넌트 목록**: 버튼(주/보조/텍스트/위험), 카드, 리스트 행, 배지(체험중·지각 동기화·취소·광고·역할), 칩(카테고리·장소·결제수단 세그먼트), 금액 텍스트(크기 3단), 수량 스테퍼, 토글, 입력 필드(텍스트·금액), 탭/세그먼트, 잠금 오버레이(체험+구독 CTA 포함), 네이티브 광고 카드 프레임, 확인 다이얼로그, 토스트/알림 카드, 캘린더 그리드, 바 차트(14일 추이·오늘 강조), 랭킹 바, 빈 상태 프레임, 동기화 pill, 하단 CTA 바.
- **접근성**: 터치 타깃 최소 48dp, 본문 대비 WCAG AA(야외 사용으로 실질 AAA 지향), 색+형태 이중 인코딩(원가율·취소 상태), 시스템 폰트 크기 1.3배까지 레이아웃 유지.
- **다크모드**: 이번 범위는 라이트 확정 + **토큰 구조만 다크 대응 가능하게** 설계(다크 시안은 선택 제출).
